# 分镜迭代任务列表

## B1  叙事结构域模型重构（未开始）
- **目标**：将 Story → Episode → Script 扩展为工业化剧本所需的 Treatment、Step Outline、Scene、Shot 等分层模型，实现数据对象对齐制作流程。
- **行动项**：
  - 新增 `story_treatments`、`story_step_outlines` 等模型，映射剧情大纲与逐场梗概，并建立与 `stories` 的版本化关联。
  - 拆分 `scripts.scenes`/`dialogues` JSON 字段为独立 `scenes`、`scene_beats`、`shots` 表，记录 INT/EXT、时间、景别、机位、镜头运动等关键元数据。
  - 调整 Service 层与 API，提供按剧集/场景/镜头级查询与写入接口，补充迁移脚本与数据兼容策略。

## B2  剧本版本与审核流程（未开始）
- **目标**：提供工业剧本常用的版本管理（Draft、Blue、Pink 等）与审校记录，使生成/人工修改可追踪。
- **行动项**：
  - 设计 `script_versions`、`scene_revisions` 表，记录版本号、颜色批次、修改说明、审批人等信息。
  - 在后台/前端增加版本比较视图，展示场景差异（新增/删除/修改）以及审阅意见。
  - 将版本信息与生成历史、提示词快照联动，支持回滚与再生成。

## B3  角色资产与关系网络强化（未开始）
- **目标**：补齐工业剧本对角色背景、阶段性弧线、关系图谱的需求，为提示词与分镜生成提供结构化输入。
- **行动项**：
  - 扩展 `story_characters` 为独立角色档案，拆分人物基础信息、外观造型、情绪/口头禅、弧线节点。
  - 建立 `character_relationships` 图结构（伙伴、对立、亲属）及变迁历史，暴露给剧本和分镜提示词。
  - 在 AI 生成流程中注入角色限制（不可变的设定、造型锚点），并设计校验规则。

## P1  提示词模板组件化（未开始）
- **目标**：沉淀对话中总结的工业级提示词结构，形成模块化模板与变量体系，减少临时拼接。
- **行动项**：
  - 新增 `prompt_templates`、`prompt_sections` 数据模型，覆盖项目类型、风格基调、场景描述、镜头语言、视觉风格、声音氛围、锚点等段落。
  - 为 Story/Episode/Scene/Shot 级实体生成对应的 `prompt_context`，自动填充模板变量并允许人工覆写。
  - 在前端提供提示词片段库与版本切换，支持从模板派生个性化提示并保存差异。

## P2  提示词执行评估闭环（未开始）
- **目标**：建立提示词 → 生成结果 → 质量评分的追踪机制，持续验证提示词有效性。
- **行动项**：
  - 设计 `prompt_runs`、`generation_feedback` 表，记录模型、参数、生成产物、人工/自动评分指标（准确度、一致性、镜头合规等）。
  - 构建对比分析工具（例如同一场景多提示词 A/B Test），输出改进提示词的建议。
  - 将评估结果回写到模板与任务看板，形成“提示词健康度”视图。

## P3  提示词权限与发布流程（未开始）
- **目标**：对提示词资产做分级管理，确保核心模板的变更可审计、可回滚。
- **行动项**：
  - 引入模板草稿/审核/发布状态机，配置不同角色的编辑与审批权限。
  - 在 agent_chats 与版本记录中联动提示词变更，自动生成更新日志。
  - 设计灰度发布策略，允许小范围验证后再推广到默认模板。

## T1  分镜提示词与上下文校准（未开始）
- **目标**：确保分镜生成时能够引用现有剧本、场景结构、舞台指示等信息，输出描述与场景一一对应。
- **行动项**：
  - 审核 `ai_service.generate_storyboard` 与计划流程的提示词，补充剧本结构（场景编号、地点、角色、对白）和剧集/故事元数据。
  - 为每个分镜帧在生成前注入真实场景摘要，以避免模板化描述。
  - 增加单元测试/快照样例，验证返回的 frames 含有差异化、且引用到正确的场景信息。

## T2  ReAct 推理机制落地（未开始）
- **目标**：让 LangGraph/ReAct 过程真正参与分镜生成与校验，而非只有提示词约束。
- **行动项**：
  - 扩展 `StoryboardReActReasoner`，在“plan → critique → finalize”节点间持久化推理痕迹，并对重复镜头、缺失字段执行自动修复。
  - 为 reasoner 输出添加结构化日志/调试开关，便于回溯每个决策步骤。
  - 建立集成测试，覆盖“仅计划”“计划+细化”“失败回退”三条路径，保证 reasoner 可控可回退。

## T3  剧本管理页面重构（未开始）
- **目标**：重新设计前端剧本管理（含分镜）体验，支持多脚本对比、分镜版本切换与编辑同步。
- **行动项**：
  - 规划信息架构：顶部摘要、剧本内容预览、分镜/规划结果、任务队列状态等模块化布局。
  - 设计并实现新的 React 组件（含场景树、镜头列表、版本/历史面板），确保局部生成后 UI 自动刷新。
  - 对接新版 API（含 `meta`、`plan`、版本信息），并补充关键交互的单元/端到端测试。

> 备注：以上任务按优先级从上到下执行，完成前可在 PR 中引用任务编号以便追踪。
