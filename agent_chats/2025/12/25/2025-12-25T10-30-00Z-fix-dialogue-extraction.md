## User Prompt

"是对白，不是旁白，你在干什么？" (It's dialogue, not narration, what are you doing?)

The user reported that TTS was reading scene descriptions/narration instead of actual character dialogues.

## Goals

1. Fix the mock script fallback to generate proper character dialogues
2. Ensure TTS reads real dialogue content like "老拐：'撑住，我们快到了。'" instead of narration like "我们正在面对：老拐紧握方向盘..."

## Changes

### `ai-pic-backend/app/services/ai_service.py`

**Root Cause**: The `_generate_mock_script()` method was wrapping scene descriptions as fake dialogues:

```python
# BUG (old code):
dialogues.append({
    "character": focus_characters[0] if focus_characters else "旁白",
    "content": f"（{dialogue_style}）我们正在面对：{description}",  # This is narration, not dialogue!
})
```

**Fix 1**: Added new `_extract_dialogues_from_summary()` method (lines 852-904):

- Extracts real character dialogues from scene summaries using regex patterns
- Matches patterns like `角色名：'对白内容'` or `角色名："对白内容"`
- Supports both Chinese and English quotation marks
- Character name restricted to 2-4 characters to avoid matching verbs like "老拐猛踩油门"
- Uses sentence boundary detection (punctuation/whitespace) to avoid false matches
- Deduplicates extracted dialogues
- Maps extracted character names to known characters from `focus_characters`

**Fix 2**: Modified `_generate_mock_script()` to use extraction:

```python
# NEW (fixed code):
extracted_dialogues = self._extract_dialogues_from_summary(
    description, idx, focus_characters
)
dialogues.extend(extracted_dialogues)
```
