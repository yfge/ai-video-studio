## Changes

### New Files Created

1. **`app/services/agent_core/quality_loop.py`** (~350 lines)
   - `ValidationLayer` enum: DETERMINISTIC, SEMANTIC
   - `FailureMode` enum: 8 failure modes (JSON_PARSE, SCHEMA_VIOLATION, etc.)
   - `ValidationResult` dataclass: Validation outcome with timing
   - `DeterministicValidator`: Fast rule-based validation
     - Pluggable rules
     - Automatic failure mode classification
     - Duration tracking
   - `SemanticValidator`: Deep LLM-based validation
     - Pluggable semantic checks
     - Exception handling as warnings
   - `TwoLayerValidator`: Combined validation pipeline
     - Configurable semantic skip on fast fail
     - `is_valid()` convenience method
   - `RepairAttempt` dataclass: Repair attempt record
   - `RepairMetrics` dataclass: Aggregated repair statistics
   - `RepairMonitor`: Success rate tracking with SLO alerts
     - Window-based rate calculation
     - Alert cooldown to prevent spam
     - Problematic pattern identification
     - Metrics by failure mode and strategy

2. **`tests/unit/services/agent_core/test_quality_loop.py`** (36 tests)
   - ValidationResult tests
   - DeterministicValidator tests (rules, error handling, classification)
   - SemanticValidator tests (checks, exception handling)
   - TwoLayerValidator tests (pipeline, skip behavior)
   - RepairMonitor tests (tracking, SLO alerts, cooldown)
   - Integration tests (full pipeline)
